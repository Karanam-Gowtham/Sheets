DSA Paradigms

How to think about a problem before you code it

Patterns tell you what shape a solution takes. Paradigms tell you how to think. This page explains the major DSA approaches, what signals point to them, and what kind of question each one answers well.

How to use this page

  • Read the clue first and try to name the paradigm before looking at the answer.
  • Use the "ask yourself" prompt to test whether the paradigm really fits.
  • Treat the examples as memory hooks, not as the full boundary of that approach.
